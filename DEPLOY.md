# Deploy: Render (API) + Vercel (client)

Stack for production:

- **MongoDB Atlas** — database
- **Render** — Express API (`server/`)
- **Vercel** — Vite/React SPA (`client/`)

Both frontend and API must be **HTTPS**. Refresh cookies use `SameSite=None; Secure` when `NODE_ENV=production`.

---

## 0. Prerequisites

- Repo pushed to GitHub
- [MongoDB Atlas](https://cloud.mongodb.com) account
- [Render](https://render.com) account
- [Vercel](https://vercel.com) account
- (Optional) KMITL SSO client at [developer.kmitl.ac.th](https://developer.kmitl.ac.th/console/sso)

---

## 1. MongoDB Atlas

1. Create a free cluster.
2. **Database Access** → create a DB user + password.
3. **Network Access** → allow `0.0.0.0/0` (or lock to Render IPs later).
4. **Connect → Drivers** → copy the URI, and set the DB name to `mood-of-the-major`:

```text
mongodb+srv://<user>:<password>@<cluster>.mongodb.net/mood-of-the-major?retryWrites=true&w=majority
```

---

## 2. Render (API)

### Option A — Blueprint (`render.yaml`)

1. Render Dashboard → **New** → **Blueprint**.
2. Connect the GitHub repo (root contains `render.yaml`).
3. Apply the blueprint. Fill every env var marked as needing a value (`sync: false`):

| Variable | Example |
|----------|---------|
| `MONGODB_URI` | Atlas URI from step 1 |
| `CLIENT_URL` | `https://your-app.vercel.app` (update after Vercel deploy) |
| `ADMIN_STUDENT_ID` | e.g. `99999999` |
| `ADMIN_EMAIL` | e.g. `99999999@kmitl.ac.th` |
| `ADMIN_PASSWORD` | strong password (not `Admin123!`) |
| `KMITL_OIDC_*` | leave empty to disable SSO, or fill for production SSO |

JWT secrets are auto-generated by the blueprint.

### Option B — Manual Web Service

| Field | Value |
|-------|--------|
| Root Directory | `server` |
| Runtime | Node |
| Build Command | `npm install` |
| Start Command | `npm start` |
| Health Check Path | `/api/health` |

Set the same env vars as in Option A, plus:

```text
NODE_ENV=production
JWT_ACCESS_SECRET=<openssl rand -hex 32>
JWT_REFRESH_SECRET=<openssl rand -hex 32>
JWT_ACCESS_EXPIRES_IN=15m
JWT_REFRESH_EXPIRES_IN=7d
```

### After API is live

1. Note the URL: `https://mood-of-the-major-api.onrender.com` (yours may differ).
2. Open `https://<api-host>/api/health` → `{ success: true, data: { status: "ok" } }`.
3. Seed admin once (Render **Shell**):

```bash
npm run seed:admin
```

4. Set `KMITL_OIDC_REDIRECT_URI` to:

```text
https://<api-host>/api/auth/kmitl/callback
```

## 3. Vercel (client)

1. Vercel → **Add New Project** → import the same GitHub repo.
2. Configure:

| Field | Value |
|-------|--------|
| Root Directory | `client` |
| Framework Preset | Vite |
| Build Command | `npm run build` |
| Output Directory | `dist` |

3. Environment variable (**Production**):

```text
VITE_API_URL=https://<api-host>
```

No trailing slash. This is baked in at **build** time — change it → redeploy.

4. Deploy. Note the URL: `https://your-app.vercel.app`.

`client/vercel.json` already rewrites all routes to `index.html` for React Router.

---

## 4. Wire the two URLs together

1. On **Render**, set:

```text
CLIENT_URL=https://your-app.vercel.app
```

Exact origin, HTTPS, no trailing slash. Then **redeploy** the API.

2. On **Vercel**, confirm `VITE_API_URL` points at the Render API. Redeploy if needed.

3. (SSO) In the KMITL developer console, add redirect URI:

```text
https://<api-host>/api/auth/kmitl/callback
```

## Troubleshooting

| Symptom | Likely fix |
|---------|------------|
| CORS error | `CLIENT_URL` must match the Vercel origin exactly |
| Login works then immediately logs out | Both sides must be HTTPS; `NODE_ENV=production` on Render |
| Frontend still hits localhost | Rebuild Vercel after setting `VITE_API_URL` |
| Atlas connection refused | Network Access / wrong URI / URL-encoded password |
| Free Render cold start | First request after idle can take ~30–60s |
| SSO fails | Redirect URI mismatch in KMITL console vs `KMITL_OIDC_REDIRECT_URI` |
