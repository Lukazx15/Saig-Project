# Deploy: Render (API) + Vercel (client)

Stack for production:

- **MongoDB Atlas** — database (user profiles only; **no passwords stored**)
- **Render** — Express API (`server/`)
- **Vercel** — Vite/React SPA (`client/`)

Auth is **KMITL SSO only** (login + registration). Both frontend and API must be **HTTPS**. Refresh cookies use `SameSite=None; Secure` when `NODE_ENV=production`.

**Current production URLs** (update if yours differ):

| Piece | URL |
|-------|-----|
| Client | https://saig-project.vercel.app |
| API | https://mood-of-the-major-api.onrender.com |

---

## 0. Prerequisites

- Repo pushed to GitHub
- [MongoDB Atlas](https://cloud.mongodb.com) account
- [Render](https://render.com) account
- [Vercel](https://vercel.com) account
- **KMITL SSO client** at [developer.kmitl.ac.th](https://developer.kmitl.ac.th/console/sso) — required (no password login)

---

## 1. MongoDB Atlas

1. Create a free cluster.
2. **Database Access** → create a DB user + password (Atlas DB credentials only — **not** app user passwords).
3. **Network Access** → allow `0.0.0.0/0` (or lock to Render IPs later).
4. **Connect → Drivers** → copy the URI, and set the DB name to `mood-of-the-major`:

```text
mongodb+srv://<user>:<password>@<cluster>.mongodb.net/mood-of-the-major?retryWrites=true&w=majority
```

If the password has special characters (`@`, `#`, `/`, etc.), URL-encode them in the URI.

Set `MONGODB_URI` only in the Render dashboard (`sync: false` in `render.yaml` — never commit the URI). If a MongoDB password was ever committed to git, **rotate it in Atlas** and update the Render secret.

App documents store `studentId`, `email`, `faculty`, `major`, `year`, `role` — not `passwordHash`. After upgrading from password-era code, run `npm run seed:admin` once so legacy `passwordHash` fields are stripped.

## 2. Render (API)

### Option A — Blueprint (`render.yaml`)

1. Render Dashboard → **New** → **Blueprint**.
2. Connect the GitHub repo (root contains `render.yaml`).
3. Apply the blueprint. Fill every env var marked as needing a value (`sync: false`):

| Variable | Example |
|----------|---------|
| `MONGODB_URI` | Atlas URI from step 1 |
| `CLIENT_URL` | `https://saig-project.vercel.app` (exact Vercel origin, no trailing slash) |
| `ADMIN_STUDENT_ID` | your 8-digit student ID (admin signs in via KMITL SSO) |
| `ADMIN_EMAIL` | e.g. `<studentId>@kmitl.ac.th` |
| `KMITL_OIDC_CLIENT_ID` | from KMITL developer console |
| `KMITL_OIDC_CLIENT_SECRET` | from KMITL developer console |
| `KMITL_OIDC_REDIRECT_URI` | `https://saig-project.vercel.app/api/auth/kmitl/callback` |

JWT secrets are auto-generated by the blueprint. `ADMIN_PASSWORD` is **not used** — remove it from Render env if it still exists.

### Option B — Manual Web Service

| Field | Value |
|-------|--------|
| Root Directory | `server` |
| Runtime | Node |
| Build Command | `npm install` |
| Start Command | `npm start` |
| Health Check Path | `/api/health` |

Set the same env vars as in Option A, plus:

```text
NODE_ENV=production
JWT_ACCESS_SECRET=<openssl rand -hex 32>
JWT_REFRESH_SECRET=<openssl rand -hex 32>
JWT_ACCESS_EXPIRES_IN=15m
JWT_REFRESH_EXPIRES_IN=7d
KMITL_OIDC_ISSUER=https://sso.kmitl.ac.th/realms/kmitl
```

### After API is live

1. Note the URL: `https://mood-of-the-major-api.onrender.com` (yours may differ).
2. Open `https://<api-host>/api/health` → `{ success: true, data: { status: "ok" } }`.
3. Seed admin once (promotes `ADMIN_STUDENT_ID` / `ADMIN_EMAIL` and strips legacy `passwordHash`):

**Render Shell** (paid plans with Shell):

```bash
npm run seed:admin
```

**Free plan (no Shell)** — run locally against Atlas:

```powershell
cd server
# Ensure server/.env has MONGODB_URI (Atlas) + ADMIN_STUDENT_ID + ADMIN_EMAIL
npm run seed:admin
```

Do not commit `.env`.

4. Set `KMITL_OIDC_REDIRECT_URI` to the **Vercel** callback (proxied to Render):

```text
https://saig-project.vercel.app/api/auth/kmitl/callback
```

Do **not** use the raw `*.onrender.com` callback in production — cookies must be set on the Vercel host.

## 3. Vercel (client)

1. Vercel → **Add New Project** → import the same GitHub repo.
2. Configure:

| Field | Value |
|-------|--------|
| Root Directory | `client` |
| Framework Preset | Vite |
| Build Command | `npm run build` |
| Output Directory | `dist` |

3. Environment variable (**Production**):

```text
VITE_API_URL=
```

Leave **empty** (or omit). The client then calls same-origin `/api/...`, and `client/vercel.json` proxies those requests to Render. That is required so httpOnly cookies (`refreshToken`, `ssoTicket`) are first-party — a cross-origin `VITE_API_URL=https://….onrender.com` breaks SSO registration and session refresh in modern browsers.

If you must set it explicitly, use the Vercel origin itself:

```text
VITE_API_URL=https://saig-project.vercel.app
```

Never point production `VITE_API_URL` at the Render hostname.

4. Deploy. Note the URL: `https://saig-project.vercel.app`.

`client/vercel.json` proxies `/api/*` → Render, then falls back to `index.html` for React Router.

---

## 4. Wire the two URLs together

1. On **Render**, set:

```text
CLIENT_URL=https://saig-project.vercel.app
```

Exact origin, HTTPS, no trailing slash. Then **redeploy** the API. A mismatch causes `Origin not allowed` / CORS errors.

2. On **Vercel**, confirm `VITE_API_URL` is empty (or the Vercel origin). Redeploy if you changed it.

3. In the KMITL developer console, set redirect URI to:

```text
https://saig-project.vercel.app/api/auth/kmitl/callback
```

Match `KMITL_OIDC_REDIRECT_URI` on Render exactly.

## Troubleshooting

| Symptom | Likely fix |
|---------|------------|
| CORS / `Origin not allowed` | `CLIENT_URL` must match the Vercel origin exactly (no trailing slash) |
| Login works then immediately logs out | Both sides must be HTTPS; `NODE_ENV=production` on Render |
| Frontend still hits localhost / onrender | Clear `VITE_API_URL` on Vercel and redeploy so calls go to `/api` |
| `No KMITL SSO registration session found` | Cross-site cookie — use same-origin `/api` proxy + Vercel redirect URI |
| Atlas connection refused | Network Access / wrong URI / URL-encoded password |
| Free Render cold start | First request after idle can take ~30–60s |
| SSO fails | Redirect URI mismatch in KMITL console vs `KMITL_OIDC_REDIRECT_URI` |
| `DEPLOYMENT_NOT_FOUND` on Vercel | Project/domain missing — redeploy client or update `CLIENT_URL` to the live domain |
| Looking for password login / `ADMIN_PASSWORD` | Removed — auth is KMITL SSO only |
